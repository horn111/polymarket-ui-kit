# Launch Playbook

## Prelaunch

- Finish README first screen.
- Publish docs and demo.
- Record three short clips: market card, orderbook, share card.
- Prepare a pinned GitHub issue for component requests.
- Ask three builders to try the registry install.

## Launch

- Post a concise X thread with visuals.
- Share in builder/dev communities.
- Submit to Polymarket Builders.
- Reply to every technical question with a snippet or docs link.

## After launch

- Ship small visible updates every 2-3 days.
- Turn each update into a before/after tweet.
- Track stars, installs, issues, and demo usage.
- Convert repeated feedback into roadmap items.
