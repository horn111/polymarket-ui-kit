# Grant Tweet Thread

## Tweet 1

I submitted Polymarket UI Kit for the Polymarket Builders program.

The thesis is simple:

Polymarket has APIs.
Builders still need a distribution UI layer.

Market cards, public hooks, Link-to-Embed Studio, share export, Builder-Code UX,
and Combo-aware components should not be rebuilt from scratch every time.

Repo: https://github.com/horn111/polymarket-ui-kit
Demo: https://polymarket-ui-kit-demo-fkan-chi.vercel.app/

## Tweet 2

The kit is read-first by design.

No hidden order placement.
No auto-injected Builder Code.
No quote submission.

Host apps keep control of CLOB/RFQ flows. The kit handles the surfaces builders
need to make markets readable, embeddable, and shareable.
