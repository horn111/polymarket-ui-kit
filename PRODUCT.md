# Product

## Register

brand

## Platform

web

## Users

The primary audience is split evenly between builders evaluating a production-ready
React kit and reviewers assessing the quality of the open-source work. A second,
equally important audience is design and motion practitioners encountering the project
as a portfolio-grade digital product.

## Product Purpose

Polymarket UI Kit provides typed React components, public data adapters, source-aware
context, embeds, and share surfaces for market applications. The project succeeds when
the same system feels credible in code review, product use, a grant evaluation, and a
cinematic portfolio presentation.

## Positioning

A category-neutral market interface system that makes probability feel physical,
legible, and supported by evidence.

## Conversion & proof

- Primary CTA: Try Studio.
- Secondary CTA: inspect the components, registry, and GitHub source.
- Ten-second memory: Make probability feel credible.
- Belief ladder: the interface is distinctive; the data remains legible; the context is
  sourced; the components are real and reusable; distribution works across React,
  iframe, PNG, SVG, and registry.
- Proof on hand: live public market data, typed packages, Storybook states, export
  routes, registry items, tests, grant documentation, and open-source history.

## Brand Personality

Engineered, tactile, assured. The product should feel like a calibrated analytical
instrument, with controlled cinematic direction and no speculative-tech spectacle.

## Anti-references

Avoid generic SaaS card grids, editorial-serif politics styling, crypto neon, terminal
brutalism, party-coded red/blue pairs, purple or violet AI palettes, chartreuse accents,
glassmorphism as a default material, and identical fade-up motion on every section.

## Design Principles

1. Make data feel physical without reducing its clarity.
2. Use motion to explain calibration, hierarchy, and state.
3. Let evidence and provenance sit beside probability.
4. Give every layer a mechanical role; decoration alone is not a role.
5. The marketing surface and component library must prove the same level of craft.

## Accessibility & Inclusion

Target WCAG AA. Preserve keyboard navigation, visible focus, screen-reader semantics,
color-independent chart cues, reduced-motion behavior, readable contrast, and layouts
that remain complete from 320 px through large desktop widths.
